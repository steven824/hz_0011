About Us

The electronic bill of lading system is a paperless "bill of lading" generated from the contract data related to the carriage of goods by sea transmitted through the Electronic data interchange system.

Transmitting a series of electronic data combined according to certain rules through the Electronic data interchange system and transferring them by password, on the one hand, it speeds up the transferability of documents, on the other hand, it also prevents the use of traditional bills of lading for maritime fraud, which is an important part of the evolution from "paper-based trade" to "paperless trade"

Essential Tech Used In Project

  • Quickly build the backend project using SpringBoot, use MVC three-layer architecture, use Oracle as database, and use Mybatis and MyBatis Plus as the data persistence framework
  • Using static template tech which is FreeMarker to solve page fast loading, and using distributed file system MiniO to store massive files
  • Use a delayed queue solution to solve the scheduled publication of report articles, use DFA algorithm to filter sensitive words in articles, and use Tess4J to recognize text in images
  • The project uses Redis for caching common data queries, and Spring Data Redis supports caching operations
  • Using RabbitMQ to achieve asynchronous task processing, improve the system's throughput of receiving requests and the response speed it brings, and enhance the user experience
