1. Exception data processing

The Exception data processing module includes: to-do tasks, processed tasks, and abnormal goods processing
1.1 to-do tasks

Click on the agent processing number on the system homepage to enter the agent task list

Alternatively, in the left navigation menu - Exception Data Processing - To-do Tasks, enter To-do Tasks

Dangerous goods verification will be made automatically by system based on the Dangerous Goods Catalogue after agent input manifest data into the system. If the manifest data screened by the system is inconsistent with the information in the Dangerous Goods Catalogue, the agent will report the manifest data to the supervisor for processing. The supervisor will log in the system for processing through the 'Agent Tasks' module. Data inconsistency mainly occurs in the following situations:

  1. It is a hazardous material, but the agent did not truthfully report it and reported it as regular goods
  2. It is a dangerous goods, and the agent has reported it, but the reported dangerous goods information will not be 100% consistent with the dangerous goods catalogue
  3. It is a dangerous goods, but there is no information about this dangerous goods in the dangerous goods catalogue

The regulatory authorities need to handle the above situation



Users can click on 【process】 to view the verification details of the manifest, specific information of the goods on the manifest, and historical processing opinions

    【Add to whitelist】：By manually determining whether the goods are dangerous or not, the user can click on the 【Add to whitelist】 button to add the goods to the white list (note: regardless of whether the goods exist in the dangerous goods catalogue, they will be classified as regular goods during verification)

    【verification history】：Different jurisdictional accounts can query the verification results of verified manifests in their respective jurisdictions. You can switch through the timeline to view the historical verification status of this checked manifest

    【recheck】：After adding the goods to the whitelist, users can click the 【recheck】 button to obtain the check result again

    【process】：According to manual judgment, the user can select 【pass】, select processing measures, fill in the processing status, and then click to confirm the successful processing of the manifest. The processed manifest information will flow to the 【processed task】 interface

1.2 Task processed

Function entry: Left navigation menu - Exception data processing - Processed tasks

The regulatory authority has the authority to query the processed manifest information

Enter the keyword 【Ship Name Voyage】 to quickly query the ship name voyage. Click on the desired 【Ship Name Voyage】 and a task list will appear on the right

Click 【view】 to view the verification details, goods information, and handling suggestions of the manifest

1.3 Handling of Abnormal Goods

Function entry: Left navigation menu - Exception data processing - Abnormal goods processing

After reporting, the suspected dangerous goods and dangerous goods in the manifest are displayed here in the form of a single item of goods information. Click on the single item of information to enter the relevant manifest information, where you can see the verification details, goods information, and handling opinions of the manifest. When handling, all dangerous goods or suspected dangerous goods in the same manifest have been processed

Enter the keyword 【Ship Name Voyage】 to quickly query the ship name and voyage. Click on the desired 【Ship Name Voyage】 and a task list will appear on the right

Recheck: Re compare with the dangerous goods catalogue and whitelist, and ensure that the processing interface is consistent with the to-do task interface

2. Data Query

2.1 view verification result

Function entry: Left navigation menu - Data query - Check verification results

All manifest information that has been verified can be viewed in the verification result viewing module, including ship information, manifest information, verification time, processing status, processing results, etc

The agent can only check the verification results of the manifest uploaded by themselves; The maritime department of the jurisdiction shall check the situation of manifest verification within the management scope of the jurisdiction; Administrators can view all manifest verification results

Exception data refers to the viewing of processing results

3. Parameters setup

Parameter setting is the basic configuration module for verification. The name of goods is verified against the correct transport name, chemical formula, alias and English name in the dangerous goods catalogue. The parameter setting module includes: dangerous goods category, dangerous goods catalogue and white list of goods
3.1 Dangerous goods category

Function entry: Left navigation menu - Parameter settings - Dangerous goods category

Establish an overall framework for the hazardous material catalog

【Add】：Click 【All】 in the tree menu on the left, and then click 【Add】 to read the category information of dangerous goods

【Edit】：Select the established dangerous goods category and click 【Edit】 to modify the name, overall description in chinese, and other information of the dangerous goods category

【Delete】：Click the 【Delete】 button to delete all dangerous goods in this category and its folder

【Add New Category】：Click 【Add】, fill in all the information, and click 【Save】

3.2 Dangerous Goods Catalogue

Function entry: Left navigation menu - Parameter settings - Dangerous goods catalogue

The dangerous goods catalogue module realizes the function of input a single piece of dangerous goods information under the subcategory, including UN dangerous goods number, correct transport name, chemical name, English name, alias, CAS number, etc., as shown in the figure below

【Search】：Enter the keywords of the goods name and click the 【Search】 button to perform a fuzzy query on it

【Index Generation】：Click the 【Index Generation】 button to make the newly added dangerous goods information effective. After any new or modified operations on this module, you must click 【Index Generation】 to take effect

【Add On Left】：Add new dangerous goods information

【Edit】：Modify dangerous goods information

【Delete, Batch Delete】：Delete dangerous goods information

【Add On Right】：Add a new dangerous goods alias, click Save, and the addition is successful

【Alias Addition】：Click 【Add】 on the right side, fill in the hazardous material alias and serial number, click 'Save', click 【Index Generation】, and the new alias is successfully added

Dangerous goods addition: Click on the left side 【Add】 to pop up the dangerous goods information entry interface. Fill in all information 【Red box required】, click Save, click on 【Index Generation】, and the addition is successful.

3.3 goods whitelist

Function entry: Left navigation menu - Parameter settings - goods whitelist

Any goods added to the dangerous goods whitelist are considered regular goods. Even if the goods are stored in the dangerous goods catalogue, they will not be classified as dangerous goods

【Search】：Fuzzy query based on correct shipping name

【Add】：Add white list goods names

【Edit】：Modify goods name

【Batch Delete】：Check the box in front of the goods name to support deleting multiple whitelist goods at once

About This System

To strengthen the safety management of dangerous goods entering and exiting Fuzhou Port, improve the supervision level of dangerous goods in Fuzhou Port, and reasonably regulate dangerous goods The approval process for reporting dangerous goods is established to prevent concealment, false reporting, or concealment of dangerous goods entering or leaving the port at various process nodes. This system is specially established to assist regulatory authorities in law enforcement work.

The system uses the FZPU platform as the information exchange center and establishes a database with a dangerous goods catalogue as the core. Compare the manifest data declared by the agent with the core database to screen out dangerous goods, suspected dangerous goods, and general goods. The system mainly includes the entry and import of manifest, manifest verification, exception handling, hazardous goods inventory management, and terminal hazardous goods qualification management.

Essential Tech Used In Project

  • The front-end adopts HTML5 and the EasyUI framework to quickly build a page architecture, and sends asynchronous requests using Ajax
  • Use Spring Boot to build the back-end framework, Mybatis and Mybatis plus as the data persistence technology, and Oracle as the data storage database
  • Use Redis to improve data interchanges efficiency. Spring Data Redis supports cache operations. Redis pipeline technology enables batch data transmission, and thread pool enables parallel processing of multiple groups of data
  • Use Elasticsearch to efficiently query massive amounts of data as well as dangerous goods verification
